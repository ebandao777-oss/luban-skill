# 快速上手

3 分钟开始用鲁班优化你的 skill。

## 1. 基本评估

输入"评估所有skills"，鲁班会：

1. 读取每个 skill 的 SKILL.md
2. 按 9 维 Rubric 逐项打分（dim1-7, dim9 结构评分 + dim8 dry_run 推演）
3. 输出每个 skill 的总分和最低维度

## 2. 优化单个 Skill

输入"优化 <skill名>"，鲁班会：

1. 设计 2-3 个测试 prompt 并展示给你确认
2. 基线评估（结构 + 效果）
3. 按短板逐轮优化（每轮 1 个维度，最多 3 轮）
4. 每轮展示改动 + 分数变化，等你确认才继续

## 3. 批量优化

输入"优化所有skills"，鲁班按基线分升序逐个处理（最弱的先修）。

## 4. 查看历史

输入"看看skill优化历史"，鲁班读取目标 skill 目录下的 `results.tsv` 展示过往评分记录。

## 常见问题

**Q: 为什么 dim8 没有实际跑测试？**
A: 默认是 Quick 模式（dry_run 推演）。要跑真实效果测试，用"完整优化"触发 Full 模式。Full 模式下鲁班会派发子 agent 独立执行 test-prompts。

**Q: 为什么改了 dim3 之后 dim2 也涨了？**
A: dim2/3/4 是相关簇——补充失败处理（dim3）会让工作流更清晰（dim2），检查点也自然增加（dim4）。这是预期行为。

**Q: 优化后分数没涨怎么办？**
A: 鲁班自动回滚并记录到 `rejected_edits.md`。可以看看被拒原因，换个方向试。

**Q: 出错了怎么办？**
A: 鲁班有 11 条异常 fallback 规则。遇 git 不可用会切 `.bak` 备份，遇子 agent 不可用会降级评分模式。错误不会静默跳过。
